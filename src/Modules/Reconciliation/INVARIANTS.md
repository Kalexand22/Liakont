# Invariants — module Reconciliation

| ID | Rule | Enforcement |
|---|---|---|
| INV-RECONCILIATION-001 | **Trois stratégies, par confiance décroissante** (F06 §7 §1) : (1) numéro de document dans le NOM DE FICHIER → confiance HAUTE ; (2) numéro dans le TEXTE du PDF → confiance HAUTE ; (3) date d'émission ET montant TTC présents, candidat UNIQUE → confiance MOYENNE. | `ReconciliationEngine.Decide` (pur) ; `DocumentNumberMatcher` (jeton délimité, anti faux-positif sur sous-chaîne). Tests `ReconciliationEngineTests` (nom, contenu, date+montant), `DocumentNumberMatcherTests`. |
| INV-RECONCILIATION-002 | **Jamais de lien automatique sous la confiance HAUTE** (décision 2026-06-02) : une confiance MOYENNE est une PROPOSITION (confirmation opérateur requise), jamais un lien automatique (un rapprochement erroné archivé en WORM serait incorrigible). | `ReconciliationEngine` renvoie `ProposeManual` (jamais `AutoLinked`) en stratégie 3 ; `ReconciliationService` n'ajoute addendum + audit QUE pour `AutoLinked`. Tests `ReconciliationEngine_DateAndAmount_ProposesManual`, `ReconciliationService_MediumConfidence_QueuesProposalWithoutAddendum`. |
| INV-RECONCILIATION-003 | **Ambiguïté (≥ 2 candidats) ⇒ NON RÉCONCILIÉ** (F06 §7 §1) : ni lien automatique, ni proposition — orphelin (file manuelle). Vaut pour l'ambiguïté de confiance haute (≥ 2 numéros) comme moyenne (≥ 2 date+montant). | `ReconciliationEngine` renvoie `NotReconciled` dès 2 candidats distincts. Tests `ReconciliationEngine_AmbiguousNumber_NotReconciled`, `…AmbiguousDateAmount_NotReconciled`. |
| INV-RECONCILIATION-004 | **Le PDF réconcilié rejoint le paquet d'archive en ADDENDUM (WORM)** — jamais de réécriture (cohérent TRK05 : ajout chaîné référencé). Vaut pour le lien automatique ET la confirmation manuelle. | `ReconciliationService` appelle `IArchiveService.AddAddendumAsync` (kind `pdf-reconcilie`) ; aucune méthode de modification du coffre. Tests d'intégration `ReconciliationService_AutoLink_AddsArchiveAddendum`, `…ManualConfirm_AddsArchiveAddendum`. |
| INV-RECONCILIATION-005 | **Tout rapprochement est journalisé sur le document émis** (F06 §7) : `DocumentReconciledAuto` (système) pour un lien automatique, `DocumentReconciledManual` (avec identité opérateur OBLIGATOIRE) pour une confirmation manuelle. Append-only (garanti par Documents). | `IDocumentReconciliationJournal` (port Documents.Contracts) ; `DocumentEvent.ReconciledAutomatically`/`ReconciledManually`. Tests `DocumentEventReconciliationTests`, intégration `DocumentReconciliationJournal_*`. |
| INV-RECONCILIATION-006 | **File d'attente persistée à trois catégories** (F06 §7 §3) : propositions en attente, PDF orphelins, documents émis sans PDF. Un PDF du pool n'est traité qu'UNE fois (idempotence). | `reconciliation.reconciliation_queue` (contrainte UNIQUE sur `pool_pdf_id`) ; `IReconciliationQueries` (3 lectures) ; `ReconciliationService` ignore un PDF déjà en file. Tests d'intégration `PostgresReconciliationQueueStore_*`, `ReconciliationService_SecondPass_SkipsAlreadyProcessed`. |
| INV-RECONCILIATION-007 | **Confirmation manuelle non rétractable** : un PDF déjà rapproché (auto ou manuel) ne peut être re-rapproché (le PDF est déjà en addendum WORM). L'identité de l'opérateur est obligatoire. | `ReconciliationQueueEntry.ConfirmManually` (lève si déjà rapproché, exige l'identité). Tests `ReconciliationQueueEntryTests`. |
| INV-RECONCILIATION-008 | **Tenant-scopé** (blueprint §7, CLAUDE.md n°9/17) : aucune opération sans tenant résolu ; pool, base et coffre routent vers le tenant courant ; aucune requête cross-tenant. | `ReconciliationService.RequireTenant` (lève si non résolu) ; `IConnectionFactory`/`ITenantContext` ; le job multi-tenant passe par `TenantJobRunner` (SOL06), jamais une boucle maison. Tests `ReconciliationService_UnresolvedTenant_Throws`. |
| INV-RECONCILIATION-009 | **Robustesse de l'extraction PDF** : un PDF malformé/non-PDF/scanné ne lève jamais (orphelin), n'interrompt pas la passe ; pas d'OCR en V1. Montants en `decimal` (CLAUDE.md n°1). | `PdfPigTextExtractor.TryExtractText` (renvoie `null` sur échec) ; `DocumentCandidate.TotalGross`/comparaisons en `decimal`. Tests `PdfPigTextExtractorTests` (texte présent, contenu non-PDF → null). |
