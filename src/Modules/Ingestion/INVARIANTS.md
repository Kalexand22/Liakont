# Ingestion Module — Invariants

| ID | Rule | Enforcement |
|---|---|---|
| INV-INGESTION-001 | La clé API d'un agent n'est JAMAIS stockée ni renvoyée en clair : seuls le `key_prefix` (public) et l'empreinte SHA-256 (`key_hash`) sont persistés. La clé complète n'existe qu'à l'émission (enregistrement/rotation) et n'est révélée qu'UNE fois | `AgentKey` (génération + hash) ; `Agent.KeyHash` ; `AgentKeyIssuedDto.FullKey` renvoyé seulement par `RegisterAgent`/`RotateAgentKey` ; `AgentSummaryDto` n'expose pas la clé ; `PostgresAgentQueries` ne SELECT pas `key_hash` |
| INV-INGESTION-002 | Le registre d'agents vit dans la base SYSTÈME (résolution clé → tenant avant tout contexte tenant). Chaque ligne porte son `tenant_id` ; toute opération de GESTION (register/revoke/rotate/list) est scopée au tenant courant | `ISystemConnectionFactory` ; `IngestionTenantScope.Require(ITenantContext)` dans chaque handler de gestion ; `GetByIdForTenantAsync` + clause `WHERE tenant_id` |
| INV-INGESTION-003 | Le `key_prefix` est unique sur toute l'instance (résolution déterministe d'une clé) | Index unique `uq_agents_key_prefix` ; collision → `ConflictException` (`PostgresAgentRegistryUnitOfWork.InsertAsync`) |
| INV-INGESTION-004 | Une clé absente/mal formée/inconnue ou d'empreinte non concordante → 401 ; une clé connue mais révoquée → 403. Au niveau de l'issue, « préfixe inconnu » et « empreinte non concordante » sont indistinguables (anti-énumération) ; le coût d'empreinte est égalisé entre ces deux chemins | `AgentAuthenticator` (vérifie l'empreinte AVANT l'état de révocation, comparaison en temps constant `AgentKey.HashesMatch` ; le chemin « préfixe inconnu » appelle `Agent.SpendKeyComparisonTime` pour égaliser le temps) ; `AgentApiAuthenticationFilter` (mappe les issues vers 401/403) |
| INV-INGESTION-005 | Une version de contrat absente, inconnue ou trop ancienne (< N-1) → 426 Upgrade Required ; la plateforme supporte N et N-1 | `AgentContractVersionPolicy.IsSupported` ; `AgentApiAuthenticationFilter` (426 avant l'authentification) |
| INV-INGESTION-006 | L'historique des heartbeats est append-only : aucun chemin d'update/delete applicatif. La rétention 90 j relève de la supervision (pas une piste d'audit légale) et est assurée hors de ce module | `IAgentRegistryUnitOfWork` n'expose que `AppendHeartbeatAsync` (insert) ; aucune méthode d'update/delete sur `agent_heartbeats` |
| INV-INGESTION-007 | La configuration renvoyée à l'agent est un DÉFAUT SÛR tant que ses sources n'existent pas : `updateRequired = false` et champs d'update `null` (registre de versions OPS07 vide) ; `extractionSchedule = null` (repli de l'agent sur sa planification locale). Aucune règle inventée | `SafeDefaultAgentConfigurationProvider` (renvoie `new AgentConfigurationDto()`) — point d'extension unique pour OPS07 et la planification pilotée |
| INV-INGESTION-008 | Un agent révoqué ne peut pas faire pivoter sa clé (on réenregistre un nouvel agent) | `Agent.RotateKey` → `ConflictException` si `IsRevoked` |
