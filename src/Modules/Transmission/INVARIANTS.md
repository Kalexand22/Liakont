# Invariants — module Transmission

| ID | Rule | Enforcement |
|---|---|---|
| INV-TRANSMISSION-001 | Le comportement du produit est piloté par les **capacités déclarées** (`PaCapabilities`), jamais par un flag produit ni un `if (pa is B2Brouter)` (blueprint.md §2 ; CLAUDE.md n°6/8/16). | `PaCapabilities` (drapeaux déclaratifs + `SupportsPaymentReport(flux)`) ; résolution du client par `PaClientRegistry` (clé de type uniquement). Tests `PaCapabilitiesTests.SupportsPaymentReport_DistinguishesDomesticAndInternationalFlux`. |
| INV-TRANSMISSION-002 | Une capacité absente retourne un **résultat TYPÉ** (`PaCapabilityNotSupportedResult`), **jamais une exception**, **jamais un blocage** du produit ; le résultat est **journalisable** en français (CLAUDE.md n°12 ; acceptance PAA01). | `PaSendResult.NotSupported` / `PaGeneratedDocument.NotSupported` ; `PaCapabilityNotSupportedResult.Create` (message FR avec nom de PA + libellé). Tests `SendPaymentReportAsync_UnsupportedFlux_ReturnsTypedCapabilityGap_NoException`, `GetGeneratedDocumentAsync_WhenRetrievalUnsupported_ReturnsTypedGap_NoContent_NoException`, `CapabilityNotSupportedResult_OperatorMessage_IsFrench_AndJournalisable`. |
| INV-TRANSMISSION-003 | La résolution d'un plug-in se fait par **registre de TYPES** (clé `PaType`), chaque type **unique** ; jamais de `if (type == …)` dans le module (PAA01 §5 ; CLAUDE.md n°16). | `PaClientRegistry` (dictionnaire insensible à la casse, garde anti-doublon au démarrage). Tests `Resolve_KnownType_ReturnsClientFromFactory`, `Resolve_IsCaseInsensitiveOnPaType`, `Constructor_DuplicateType_Throws`, `IsRegistered_And_RegisteredTypes_ReflectFactories`. |
| INV-TRANSMISSION-004 | Un type de PA **inconnu** est une **erreur de configuration bloquante** : la résolution lève un message opérateur français, elle ne retourne **jamais `null`** ni ne passe en silence (« bloquer plutôt qu'envoyer faux » — CLAUDE.md n°3). | `PaClientRegistry.Resolve` lève `InvalidOperationException` (type inconnu + tenant + plug-ins disponibles). Tests `Resolve_UnknownType_Throws_WithFrenchMessage_NeverReturnsNull`, `EmptyRegistry_Resolve_Throws_AucunDisponible`. |
| INV-TRANSMISSION-005 | L'abstraction n'expose **aucun type HTTP** et le module ne référence **aucune PA concrète** (`Liakont.PaClients.*`) ni un autre **module métier** (F05 §6 ; module-rules §3 ; CLAUDE.md n°14/16). | Trois niveaux d'enforcement dans `TransmissionBoundaryTests` : (1) **NetArchTest** `Abstraction_HasNoHttpType` — pas de `System.Net.Http` dans les types compilés ; (2) **réflexion sur assemblies référencées** (`GetReferencedAssemblies`) — `Contracts_/Infrastructure_DoesNotReferenceAnyConcretePaPlugin` et `Contracts_DoesNotReachIntoAnotherBusinessModule` ; (3) **scan déclaratif des `.csproj`** — `ModuleCsprojs_DoNotDeclareForbiddenProjectReferences` détecte une `ProjectReference` interdite même si elle n'est pas encore exercée par du code. |
| INV-TRANSMISSION-006 | Les flux **10.4 (domestique)** et **10.2 (international)** sont **distincts** : un type de flux porté par la requête + deux capacités séparées (F01-F02 §1 ; acceptance PAA01). | `PaymentReportFlux` (Domestic/International) ; `PaCapabilities.SupportsDomesticPaymentReporting` ET `SupportsInternationalPaymentReporting` ; `PaymentReportPeriod.Flux`. Test `SupportsPaymentReport_DistinguishesDomesticAndInternationalFlux`. |
| INV-TRANSMISSION-007 | `PaAccountDescriptor` ne transporte **aucun secret en clair** (clé API, mot de passe) — les secrets sont chiffrés par tenant et résolus par le plug-in (CLAUDE.md n°10/18). | Conception : `PaAccountDescriptor.Settings` = configuration **non sensible** uniquement (account id, URL). Opposable en review (P1). |
| INV-TRANSMISSION-008 | **Aucune règle fiscale inventée** : les valeurs de réglage (`PaTaxReportSetting`/`Request`) et de période (`PaymentReportPeriod`) proviennent du paramétrage du tenant (CFG02), jamais du code ; aucun **montant** n'est porté en flottant (CLAUDE.md n°1/2). | DTOs neutres (pas de cadence/seuil/régime en dur) ; aucun champ `float`/`double` (montants éventuels conservés dans `RawResponse`). Opposable en review (P1). |
