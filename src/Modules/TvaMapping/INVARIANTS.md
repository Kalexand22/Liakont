# Invariants — module TvaMapping

| ID | Rule | Enforcement |
|---|---|---|
| INV-TVAMAPPING-001 | Une catégorie de TVA n'existe que dans la liste UNCL5305 sourcée F03 §2.1 (S, AA, AAA, Z, E, AE, G, K, O) ; aucune n'est inventée (CLAUDE.md n°2). | `VatCategory` (enum partagé Agent.Contracts) ; `VatCategoryParser.Parse` rejette tout code hors liste ; `MappingTableValidator` rejette une catégorie non définie au chargement. |
| INV-TVAMAPPING-002 | Catégorie E (exonéré) exige un code VATEX quel que soit le mode de taux ; un taux fixe non nul sur E est rejeté (une exonération est à 0 %) — F03 §2.2. | `MappingTableValidator.Validate` ajoute une violation ; `MappingTable.Create`/`Reconstitute` lèvent `InvalidMappingTableException`. |
| INV-TVAMAPPING-003 | Au plus une règle par couple (code régime source, part) — item TVA01 §3. Les flags source ne créent pas de 2ᵉ règle : ils restreignent la règle unique (non-correspondance → comportement par défaut `block`). | `MappingTableValidator` (HashSet sur (code, part), hors flags) + contrainte `uq_mapping_rules_regime_part` en base. |
| INV-TVAMAPPING-004 | Le taux est `decimal` exact, jamais flottant (CLAUDE.md n°1). Mode `Fixed` ⇒ valeur ≥ 0 présente ; mode `ComputedFromSource` ⇒ valeur absente. | Colonne `rate_value numeric` ; `MappingTableValidator.ValidateRate`. |
| INV-TVAMAPPING-005 | La validation structurelle s'applique à l'écriture ET au chargement ; une table invalide lève une exception (message opérateur français + action), jamais de comportement silencieux (CLAUDE.md n°3). | `MappingTable.Create` (écriture) et `MappingTable.Reconstitute` (chargement, via `TvaMappingMaterializer`) appellent `MappingTableValidator.EnsureValid`. |
| INV-TVAMAPPING-006 | Une table sans `validatedBy`/`validatedDate` est chargeable mais « NON VALIDÉE » ; cet état est exposé. | `MappingTable.IsValidated` (les deux champs requis) ; `MappingTableDto.IsValidated`. |
| INV-TVAMAPPING-007 | Régime non listé ⇒ comportement par défaut `block` (jamais d'envoi à l'aveugle) — F03 §4.1. | `MappingDefaultBehavior` (seule valeur `Block`) ; appliqué par le moteur TVA02. |
| INV-TVAMAPPING-008 | Toute lecture/écriture est scopée par `company_id` ; jamais de requête cross-tenant (CLAUDE.md n°9/17). | `WHERE company_id = @CompanyId` (UoW + queries + materializer) ; `uq_mapping_tables_company`. |
| INV-TVAMAPPING-009 | Aucune donnée client embarquée : les noms de flags source (ex. `RegimeMarge`) sont du paramétrage tenant, jamais codés dans le produit (CLAUDE.md n°7). | `SourceFlags` = dictionnaire générique `string → string` ; aucun nom de flag en dur dans le code. |
| INV-TVAMAPPING-010 | Le moteur de mapping (TVA02) est PUR et SANS ÉTAT : il ne consulte que la table du tenant passée en argument (isolation tenant structurelle, INV-008). Une part non couverte ou dont les flags requis ne sont pas satisfaits = `block`, jamais de mapping deviné (INV-007, CLAUDE.md n°2/3). Tout mapping réussi produit une `MappingTrace` (version, validateur, état de validation, règle appliquée, entrée, résultat). | `TvaMapper.Map(table, request, mappedAt)` (statique, sans champ) ; `MappingResult.Blocked` sur absence de règle / flags non satisfaits ; `MappingResult.Mapped` porte une `MappingTrace` non nulle ; couvert par `TvaMapperTests`. |
| INV-TVAMAPPING-011 | Le matching est EXACT sur `(code régime source, part)` : aucun joker. Le `"*"` de l'exemple pré-v6 de F03 §4.1 est obsolète et REFUSÉ à la validation (piège silencieux : table acceptée puis sur-blocage des frais) ; la couverture des frais s'exprime par une règle explicite par régime — F03 §3 « régime par régime » (amendement F03 §4.1 du 2026-06-04). | `TvaMapper.Map` compare par `string.Equals(..., StringComparison.Ordinal)` ; `MappingTableValidator` ajoute une violation si `SourceRegimeCode.Trim() == "*"` ; tests `Create_Wildcard_Code_Throws` + `Map_FraisOfUnmappedRegime_IsBlocked`. |
| INV-TVAMAPPING-012 | La détection de couverture (TVA03) est PURE et SANS ÉTAT : `MappingCoverageAnalyzer` ne consulte que ses arguments (aucune règle fiscale inventée, CLAUDE.md n°2). Couverture évaluée au **grain du code** (un code est couvert dès qu'une règle le référence, toutes parts confondues — les métadonnées observées ne portent pas la part ; le contrôle fin `(code, part)` reste au moteur, INV-007), comparaison **EXACTE** `Ordinal` cohérente avec INV-011 (une casse différente est signalée absente, pas faussement couverte). Double clé tenant : slug (régimes observés, base système) + `company_id` (table, base tenant), jamais cross-tenant (INV-008, CLAUDE.md n°9/17). Rapport recalculé à la demande (toujours à jour après push/modification, F03 §4.3). | `MappingCoverageAnalyzer.Analyze(observed, table)` (statique, `HashSet<string>(StringComparer.Ordinal)`) ; `GetMappingCoverageReportHandler` résout slug via `ITenantContext` + `company_id` via `ICompanyFilter` et lit via `ISourceTaxRegimeQueries`/`ITvaMappingQueries` ; tests `MappingCoverageAnalyzerTests` + `GetMappingCoverageReportHandlerTests`. |
