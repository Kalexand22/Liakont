# Invariants — module Archive

| ID | Rule | Enforcement |
|---|---|---|
| INV-ARCHIVE-001 | Le coffre est **WORM** : aucune méthode de modification ou de suppression d'un objet/d'une entrée existante (CLAUDE.md n°4). L'écriture est write-once (idempotente pour un contenu identique, conflit sinon) et **atomique** : un crash en cours d'écriture ne laisse jamais de fichier partiel au chemin canonique. | `IArchiveStore` n'expose que `WriteAsync`/`ExistsAsync`/`ReadAsync` (aucun delete/update) ; `FileSystemArchiveStore` écrit dans un fichier temporaire puis `File.Move` atomique + lecture seule ; `S3ArchiveStore` lève `ArchiveWriteConflictException` sur réécriture d'un contenu différent ; table `documents.archive_entries` WORM par triggers (`V005`). Tests `ArchiveStoreContractTests`, `FileSystemArchiveStore_Write_DifferentContentSamePath_ThrowsWormConflict`, `S3ArchiveStore_…ThrowsWormConflict`, `ArchiveEntry_Update_IsRejectedByWormTrigger`, `ArchiveEntry_Delete_IsRejectedByWormTrigger`. |
| INV-ARCHIVE-002 | **Chaîne de hashes par tenant** (F06 ; TRK05 §3) : `chain_hash(N) = SHA256(chain_hash(N-1) + entry_hash(N))`, genèse sur précédent vide. Toute altération d'une pièce casse la chaîne **à partir de ce point**. Les métadonnées d'audit (`mappingTrace`, pièces absentes + motifs, identité du document) vivent dans le fichier de contenu **haché** `archive-metadata.json` (couvert par `package_hash`) : les altérer casse aussi la chaîne. Le `manifest.json` (non haché) n'est qu'un index ; ses champs porteurs d'intégrité (`package_hash`, `chain_hash`, empreintes des fichiers) sont vérifiés contre la base et le contenu réel. | `HashChain.Next` (pur) ; `PostgresArchiveEntryStore.ReserveAsync` dérive le maillon sous verrou ; `ArchivePackageBuilder` place les métadonnées dans `archive-metadata.json` ; vérification par recalcul propagé dans `ArchiveService.VerifyTenantChainAsync`. Tests `HashChainTests`, `VerifyTenantChain_DetectsContentAlteration_AndCascades`, `VerifyTenantChain_DetectsMetadataAlteration`, `VerifyTenantChain_DetectsFileAlteration_OnRealStore`. |
| INV-ARCHIVE-003 | **Addenda chaînés** (TRK05 §4-§5) : une pièce ajoutée a posteriori (tax-report, PDF réconcilié) ne réécrit JAMAIS le paquet — nouveau fichier + manifest-addendum, chaîné sur la chaîne principale. Altérer un addendum casse la chaîne. | `ArchiveService.AddAddendumAsync` (nom de fichier séquencé anti-collision sous verrou, `manifest-addendum-NNN.json`) ; `entry_hash` d'addendum = empreinte du CONTENU (indépendante du nom). Tests `AddAddendum_ChainsOnTheDocumentPackage`, `VerifyTenantChain_DetectsAddendumAlteration`, `PackageThenAddendum_ChainsAndVerifiesIntact`. |
| INV-ARCHIVE-004 | **Empreinte de paquet** = SHA-256 déterministe, indépendante de l'ordre, des fichiers de CONTENU (le `manifest.json` en est exclu pour éviter la circularité, puisqu'il porte l'empreinte et le `chain_hash`). | `PackageHasher.Compute` (tri ordinal par nom) ; `ArchivePackageBuilder` exclut le manifest du calcul. Tests `PackageHasherTests`, `ArchivePackageBuilderTests`. |
| INV-ARCHIVE-005 | **Ordonnancement strict + reprise sûre (idempotence)** : les ajouts d'un tenant sont SÉRIALISÉS (`pg_advisory_xact_lock`) ; sous ce verrou on vérifie l'idempotence par `package_path`, on lit la tête, on dérive le maillon et un `archived_utc` STRICTEMENT croissant tronqué à la µs — SANS aucune E/S coffre dans la transaction. Le service écrit les fichiers de contenu AVANT la réservation et le `manifest.json` APRÈS commit, dérivé déterministiquement de la ligne committée : une reprise après échec partiel rejoue à l'identique (write-once idempotent), jamais d'orphelin bloquant (critique sur S3 Object Lock). Un ré-archivage du même document est un **no-op**. | `PostgresArchiveEntryStore.ReserveAsync` (`TransactionScope` + advisory lock + `SELECT … WHERE package_path` + `archived_utc` tronqué µs) ; ordre écriture/réservation/manifest dans `ArchiveService`. Tests `ArchivedUtc_IsStrictlyIncreasing_AcrossEntries`, `PackageThenAddendum_ChainsAndVerifiesIntact`, `ArchiveIssuedDocument_IsIdempotent_OnReplay`. |
| INV-ARCHIVE-006 | **Intégrité indépendante du backend** : la chaîne de hashes est l'intégrité de RÉFÉRENCE ; le verrou natif du backend (S3 Object Lock) n'est appliqué qu'EN PLUS, piloté par la **capacité déclarée** du store, jamais par un `if (store is S3)` (blueprint §6 ; CLAUDE.md n°14). | `ArchiveStoreCapabilities` ; `S3ArchiveStore` passe `applyObjectLock = Capabilities.SupportsObjectLock` ; le module ne connaît que `IArchiveStore`. Tests `S3ArchiveStore_Write_AppliesObjectLock_WhenCapabilityDeclared` / `…DoesNotApply…WhenCapabilityAbsent`. |
| INV-ARCHIVE-007 | **Tenant-scopé** (blueprint §7) : aucune opération sans tenant résolu ; le coffre est rooté sur le tenant courant et la base route vers la base du tenant courant. | `ArchiveService.RequireTenant` (lève si non résolu) ; `IConnectionFactory` (tenant courant) ; chemins du coffre préfixés par le slug de tenant assaini. Tests `ArchiveIssuedDocument_UnresolvedTenant_Throws`. |
| INV-ARCHIVE-008 | **Pièces présentes/absentes tracées** (TRK05 §2) : la facture PA et le bordereau source ne sont joints que si la capacité le permet ; une absence porte un MOTIF obligatoire (jamais silencieuse). Anti path-traversal sur tous les segments de chemin. | `ArchivePackageBuilder` (pièces optionnelles + `absentPieces` avec motif) ; `ArchiveService.ValidatePackageRequest` (motif obligatoire si pièce nulle) ; `ArchivePackageLayout.SanitizeSegment`. Tests `BuildPackageContent_WithoutOptionalPieces_RecordsAbsenceWithReason`, `ArchiveIssuedDocument_AbsentPieceWithoutReason_Throws`, `ArchivePackageLayoutTests`, `FileSystemArchiveStore_Write_PathTraversal_IsRejected`. |
| INV-ARCHIVE-009 | **Montants en `decimal`** dans le rendu lisible, jamais flottant (CLAUDE.md n°1). Rendu lisible AUTONOME (art. 289 V CGI) : HTML statique sans ressource externe, échappement anti-injection, accents UTF-8 préservés. | `ArchiveReadableDocument`/`ArchiveReadableLine`/`ArchiveVatBreakdownLine` en `decimal` ; `ReadableDocumentRenderer` (CSS en ligne, échappement des 5 caractères significatifs). Tests `ReadableDocumentRendererTests`. |
